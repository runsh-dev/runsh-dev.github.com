01

Shipped this quarter

The accent moved from cyan-teal (#33A6B8) to plum (#C56473), placing it in the same warm family as the neutral scale. The neutrals migrated from cool to 60% warmth — paper, not concrete. The 30-token surface is now stable.

The Components and Snippets sections are aligned with the apps/web implementation: buttons use accent/8 fill with a 30%-opacity border, inputs use the full --field-* contract, comment bubbles use the asymmetric corner radius.

02

Color & type

Accent: plum #C56473 (dark mode #E095A4). Semantic colors come from traditional Japanese hues: 縹 (info), 若竹 (success), 朽葉 (warning), 蘇芳 (error).

Neutrals n-1 → n-10 (warm scale, 60% warmth)

Three font families: sans / serif / mono. Each carries a full CJK fallback chain. CJK never gets synthetic bold — font-medium (500) is the ceiling.

Sans: Inter → PingFang SC → Noto Sans SC. Serif: Noto Serif CJK SC → SongTi SC → Georgia.

Subtraction isn’t doing less. It is each addition having been argued for.
03

Next quarter

Split out a print-kit token layer. The current tokens use OKLCH and color-mix — fine for browser print, but server-side renderers (e.g. weasyprint) need a hex-mirror version.

Reorganize SKILL.md and CHEATSHEET. Restructure as “core + kits,” spelling out what each kit forbids (e.g. print bans rgba and demands explicit page-break rules).

More print samples. We have a résumé and this report. Up next: a letter, a multi-page portfolio, and a landscape slide deck.
